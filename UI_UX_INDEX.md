
## üí° Best Practices

### When Creating Components
1. Use design tokens (not hardcoded values)
2. Follow 8px grid system
3. Include all states (default, hover, active, disabled, focus)
4. Ensure 4.5:1 contrast ratio minimum
5. Add keyboard navigation support
6. Include ARIA labels
7. Test with screen reader
8. Verify on mobile
9. Add loading states
10. Include error states

### When Making Changes
1. Document in appropriate file
2. Update visual guide if needed
3. Test accessibility
4. Verify cross-browser
5. Check mobile responsiveness
6. Update success metrics
7. Get user feedback

### When Planning Features
1. Review design principles
2. Check efficiency recommendations
3. Consider accessibility first
4. Plan for mobile
5. Define success metrics
6. Create test plan
